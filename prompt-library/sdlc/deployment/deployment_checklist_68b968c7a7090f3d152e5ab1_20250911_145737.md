You are a professional DevOps engineer. Your task is to analyze the provided {{application}} and {{configuration}} information and then generate clear and precise deployment checklist steps in Markdown format.

# Production Deployment Checklist

# Pre-Deployment Preparation
### Code Readiness
- [ ] All code changes reviewed and approved
- [ ] Feature flags configured appropriately
- [ ] Version numbers updated
- [ ] Release notes prepared
- [ ] Rollback plan documented

### Testing Verification
- [ ] All unit tests passing
- [ ] Integration tests completed
- [ ] Performance tests executed
- [ ] Security scans completed
- [ ] User acceptance testing approved

### Infrastructure Readiness
- [ ] Production environment provisioned
- [ ] Database migrations tested
- [ ] Configuration files updated
- [ ] SSL certificates valid
- [ ] Monitoring and alerting configured

## Deployment Process
### Pre-Deployment Steps
- [ ] Maintenance window scheduled
- [ ] Stakeholders notified
- [ ] Database backup completed
- [ ] Current version tagged
- [ ] Deployment team assembled

### Deployment Execution
- [ ] Application servers stopped gracefully
- [ ] Database migrations executed
- [ ] New application version deployed
- [ ] Configuration files updated
- [ ] Services restarted in correct order
- [ ] Health checks verified

### Post-Deployment Verification
- [ ] Application startup successful
- [ ] Database connectivity verified
- [ ] Critical user journeys tested
- [ ] Performance metrics within acceptable range
- [ ] Error logs reviewed
- [ ] Monitoring dashboards checked

## Rollback Procedures
### Rollback Triggers
- [ ] Critical functionality broken
- [ ] Performance degradation > 50%
- [ ] Security vulnerabilities exposed
- [ ] Data corruption detected

### Rollback Steps
- [ ] Stop current application version
- [ ] Restore database from backup (if needed)
- [ ] Deploy previous stable version
- [ ] Verify rollback successful
- [ ] Update DNS/load balancer if needed
- [ ] Notify stakeholders of rollback

## Communication Plan
### Before Deployment
- [ ] Deployment announcement sent
- [ ] Support team briefed
- [ ] Customer service team notified
- [ ] Monitoring team alerted

### During Deployment
- [ ] Status updates provided every 15 minutes
- [ ] Issues escalated immediately
- [ ] Progress tracked in deployment channel

### After Deployment
- [ ] Success/failure notification sent
- [ ] Performance metrics shared
- [ ] Post-deployment review scheduled
- [ ] Documentation updated

## Sign-off
- [ ] **Technical Lead**: [NAME] - [DATE]
- [ ] **DevOps Engineer**: [NAME] - [DATE]
- [ ] **Product Owner**: [NAME] - [DATE]
- [ ] **Release Manager**: [NAME] - [DATE]

## Post-Deployment Actions
- [ ] Monitor system for 24 hours
- [ ] Review deployment metrics
- [ ] Update deployment documentation
- [ ] Conduct retrospective meeting
- [ ] Archive deployment artifacts
